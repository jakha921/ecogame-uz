# Рекомендации по дальнейшему развитию EcoGame

## 1. Функциональные улучшения

### 1.1 Расширение контента
- **300+ вопросов** — текущая база (150 вопросов) покрывает базовый уровень; расширение до 300+ позволит реализовать персонализированный подбор вопросов и избежать повторений при многократных прохождениях.
- **Видеообъяснения** — добавить короткие видеоролики (15–30 сек) к сложным вопросам вместо текстовых объяснений.
- **Вопросы с изображениями** — фото экосистем, видов загрязнений, редких животных для визуального обучения.
- **Перевод на русский и каракалпакский** — расширение охвата аудитории в Узбекистане.

### 1.2 Социальные функции
- **Мультиплеер** — режим реального времени «викторина против друга» через WebSocket/Django Channels.
- **Telegram-бот** — ежедневная викторина в Telegram с уведомлениями и напоминаниями.
- **Система друзей** — сравнение очков с друзьями, приглашения.
- **Командные соревнования** — классы/группы могут соревноваться между собой.

### 1.3 Геймификация
- **Daily streak** — уведомления о разрыве серии (сейчас счётчик есть, но нет уведомлений).
- **Сезонные ивенты** — специальные наборы вопросов к датам (22 апреля — День Земли, 5 июня — Всемирный день охраны окружающей среды).
- **NFT-достижения / сертификаты** — PDF-сертификат о прохождении уровня с QR-кодом для публикации.
- **PWA (Progressive Web App)** — офлайн-доступ к образовательным статьям и уже загруженным вопросам.

### 1.4 Мобильное приложение
- React Native с переиспользованием бизнес-логики (API, stores) — 70% кода можно перенести из текущего React фронтенда.

## 2. Технические улучшения

### 2.1 Производительность
- **Redis кэш** — кэшировать частые запросы: список вопросов, лидерборд, EcoFacts. Текущее время отклика 50–200ms можно сократить до 5–10ms.
- **CDN для медиа** — изображения достижений и аватары через S3/CloudFront вместо Django MEDIA_ROOT.
- **Pagination в questions API** — при 300+ вопросах cursor-based пагинация вместо загрузки всего набора.

### 2.2 Инфраструктура
- **CI/CD pipeline** — GitHub Actions: автозапуск pytest + eslint + docker build при каждом PR; деплой в production только после прохождения всех проверок.
- **E2E тесты** — Playwright тесты для критического пути: регистрация → квиз → результаты → лидерборд.
- **Мониторинг** — Sentry для отслеживания ошибок; Grafana + Prometheus для метрик (запросы/сек, время отклика БД).
- **Горизонтальное масштабирование** — Gunicorn workers + Nginx load balancing при росте аудитории.

### 2.3 Качество кода
- **Удаление legacy моделей** — `Level`, `GameSession`, `GameProgress` сохранены для целостности миграций, но не используются. После создания чистой миграции squash — удалить.
- **OpenAPI/Swagger документация** — `drf-spectacular` для автогенерации API-документации.
- **Type stubs для Django** — `django-stubs` + `mypy` для статической типизации бэкенда.

## 3. Безопасность

- **Rate limiting** — ограничение запросов на `/auth/login/` и `/auth/google/` (сейчас нет защиты от брутфорса). Рекомендуется `django-ratelimit`: 5 попыток/минута.
- **CAPTCHA** — для формы регистрации при подозрительной активности.
- **CSP (Content Security Policy)** — ограничить источники скриптов и стилей через HTTP-заголовки.
- **Аудит зависимостей** — регулярный `uv audit` / `npm audit` для проверки уязвимостей в пакетах.
- **Email верификация** — сейчас email не подтверждается при регистрации; рекомендуется добавить для борьбы со спамом.
- **2FA** — двухфакторная аутентификация для административной панели.

## 4. Контент и партнёрства

- **Партнёрство с Министерством экологии Узбекистана** — официальная верификация вопросов и контента.
- **Интеграция с учебными программами** — адаптация вопросов к школьной программе по экологии (6–11 классы).
- **Пользовательский контент** — механизм предложения новых вопросов сообществом с модерацией администратора.

## 5. Известные ограничения текущей версии

| Ограничение | Описание | Приоритет устранения |
|-------------|----------|---------------------|
| Legacy модели | `Level`, `GameSession`, `GameProgress` в `game/models.py` не используются | Средний |
| Daily streak без уведомлений | Счётчик ведётся, но нет push/email напоминаний | Высокий |
| Нет email верификации | При регистрации email не подтверждается | Средний |
| SQLite в dev | В production используется PostgreSQL, но тесты гоняются на SQLite — возможны расхождения | Низкий |
| Нет rate limiting | Login и Google Auth endpoints уязвимы к перебору | Высокий |
| Одновременные сессии | Пользователь может открыть несколько вкладок с одной сессией квиза | Низкий |
