## Слайд 7: Модель данных (ER-диаграмма)

### Ключевые сущности (11 таблиц):

```
Player ──────────── GameSession
  │                      │
  ├── GameProgress        └── ActionLog
  │     (air, water,              │
  │      soil, bio)         EcoAction
  │
  ├── PlayerAchievement
  │         │
  │    Achievement
  │
  └── LeaderboardEntry (денормализованная)
       ↑ обновляется через Django signals
```

**Архитектурное решение**: `LeaderboardEntry` — денормализованная таблица для O(1)-чтения рейтинга (без GROUP BY по всей таблице игроков)
